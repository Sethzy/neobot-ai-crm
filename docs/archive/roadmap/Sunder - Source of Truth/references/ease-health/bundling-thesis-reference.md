# Ease Health — Bundling Thesis Reference

> **Added:** 2026-03-03
> **Relevance:** Product vision — validates Sunder's bundling strategy for real estate

---

## Context

Ease Health ($41M Series A led by a16z, March 2026) is building the "AI-native operating system for behavioral health." Their approach: consolidate 13+ fragmented services into a single platform, then layer AI on top as a retention mechanism.

This is the same structural thesis behind Sunder for real estate.

## The Toast Playbook (@signulll)

> "In specific verticals where incumbent software is fragmented enough to allow rip & replace, you'll likely see more companies running the Toast playbook — which is to own the system of record, then make every AI feature a retention mechanism rather than a standalone product."

Key insight: In the AI era, the two things that matter are **owning the underlying context** and **deploying intelligence on top of it**. Most AI startups deploy intelligence on someone else's platform (easier to sell), but you can't build defensible workflows or redesign anything that way.

## Sunder Parallel

Sunder runs the same playbook for solo real estate agents in Singapore:

| Ease Health (Behavioral Health) | Sunder (Real Estate) |
|---|---|
| CRM | CRM (contacts, deals, activities) |
| EHR | Client memory files (USER.md, MEMORY.md) |
| Scheduling | Cal.com integration |
| AI Scribe | Conversation-to-CRM extraction |
| Patient Engagement | Follow-up automation, WhatsApp/email |
| Data Platform | Supabase + knowledge base |
| Compliance Tools | Safety model (approval gates) |
| VOIP | Channel integrations (WhatsApp, email) |

The wedge is identical: fragmented incumbent tools, relationship-driven workflows, heavy admin load, and practitioners who don't want to manage a tool stack.
